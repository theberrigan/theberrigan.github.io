Криптография

  • Абсолютно стойкий шифр
plaintext - исходное сообщение. ciphertext - зашифрованное сообщение. encryption - процесс шифрования. decryption - процесс расшифровки. C = E(K, P) - зашифрованное сообщение C, полученное в процессе шифрования E из исходного сообщения P с использованием ключа K. P = D(K, C) - исходное сообщение P, полученное в процессе расшифровки D из зашифрованного сообщения C с использованием ключа K. frequency analysis - способ взлома шифра за счёт анализа встречаемости символа в алфавите (например, в англ. языке самая встречаема буква - E). substitution (подстановка) - подстановка одного элемента (буквы или последовательности бит) на место другого. permutation - частный случай подстановки, когда каждому подставляемому символу соответсвует уникальный заменяемый символ. Пример: ABC -> DEE - обычная подстановка (substitution). ABC -> DEF - permutation. mode of operation - сам алгоритм шифрования, работающий на основе permutation. Шифрование работает на основе двух компонентов permutation и mode of operation. Не все пермутации безопасны. Безопасными можно считать пермутации (secure permutation), удовлетворящие условиям: 1. Пермутация должна выполняться с использованием ключа. 2. Одной пермутации должен соответствовать только один (уникальный) ключ. 3. Результат пермутации должен выглядеть произвольным, не должно быть видно закономрностей. one-time pad шифрование: 1. Создать рендомный ключ K такой же длины, как и исходное сообщение P 2. Получить шифр C = P XOR K 3. Расшифровка: P = C XOR K При one-time pad шифровании каждый раз нужно использовать новый ключ, поскольку С1 XOR C2 = P1 XOR P2, если использовался один и тот же ключ при шифровании P1 и P2. То есть хакер может узнать XOR-разницу между двумя сообщениями С1 и C2. ! XOR случайной строки с осмысленной строкой даёт случайную строку
